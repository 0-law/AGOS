# AGOS v2.2 CSV字段规范

## 固定字段

```csv
任务ID,股票代码,股票名称,所属方向,当日表现,核心逻辑浓缩,证据类型,舆情强度,情绪强度,兑现路径,验证点,负面触发,强度,状态分类,写入状态,写入时间,失败原因
```

## 字段定义

| 字段 | 含义 | 规则 |
|---|---|---|
| 任务ID | 个股任务编号 | 排第一列；格式固定为 `ID-YYYYMMDD-序号`，如 `ID-20260626-1` |
| 股票代码 | 股票代码 | 优先补全；无法确认时写 `未提供` |
| 股票名称 | 股票名称 | 必填 |
| 所属方向 | 当日主线方向 | 无法归类时写 `其他` |
| 当日表现 | 涨停/连板/涨幅/热榜/异动/未提供 | 如实填写 |
| 核心逻辑浓缩 | 个股核心逻辑 | 不超过 80 个汉字。尽量饱满，不可因"浓缩"而省略已知的有效信息（产业链地位、催化事件、订单/业绩等），逻辑简单的不强行凑字 |
| 证据类型 | 证据标签 | 多项用 `；` 分隔 |
| 舆情强度 | 市场传播强度 | `强/中/弱/负面/未体现` |
| 情绪强度 | 盘面攻击强度 | `强/中/弱/退潮/未体现` |
| 兑现路径 | 兑现链路 | 如涨价/订单/产能/业绩/政策/客户/事件 |
| 验证点 | 后续验证信息 | 次日或后续需确认 |
| 负面触发 | 事实触发项 | `澄清/减持/低占比/仅送样/仅预研/未产品化/业绩不及预期/退潮/与公告冲突/无` |
| 强度 | 逻辑强度 | `S/A/B/C/R` |
| 状态分类 | 统一状态判断 | `利好/利空/观察` |
| 写入状态 | 写入状态 | 放最后；默认写 `未写入` |
| 写入时间 | 写入时间 | 放最后；默认留空 |
| 失败原因 | 失败原因 | 放最后；默认留空 |

## 一致性要求

- `强度` 仅表示逻辑强度。
- `状态分类` 仅允许 `利好/利空/观察`。
- `任务ID` 必填，位于第一列，格式固定为 `ID-YYYYMMDD-序号`。
- `写入状态` 默认写 `未写入`。
- `写入时间` 默认留空。
- `失败原因` 默认留空。
- 股票代码优先补全；无法确认时写 `未提供`。
