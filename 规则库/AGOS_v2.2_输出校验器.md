# AGOS v2.2 输出校验器

## 1. 工程边界

检查：

1. 是否输出买卖建议
2. 是否输出目标价
3. 是否输出仓位建议
4. 是否编造股票代码
5. 是否编造公司业务
6. 是否把小作文当事实

## 2. 三维判断

检查：

1. 是否区分逻辑强度、舆情强度、情绪强度
2. 逻辑强度是否只使用 `S/A/B/C/R`
3. 舆情强度是否只使用 `强/中/弱/负面/未体现`
4. 情绪强度是否只使用 `强/中/弱/退潮/未体现`

## 3. 状态分类

检查：

1. 是否所有股票统一进入 `个股总表`
2. 是否使用 `状态分类`
3. `状态分类` 是否只使用 `利好/利空/观察`
4. 正向信息是否标为 `利好`
5. 负向信息是否标为 `利空`
6. 小作文、映射、待验证信息是否标为 `观察`

## 4. 单篇文章映射

检查：

1. 单篇文章或无个股文章是否先做文章价值评级
2. 文章评级是否只使用 `V3/V2/V1/V0/R`
3. 是否仅在高利好或高利空情况下补个股
4. 小利好、小利空、行业压力再确认、弱催化、纯情绪波动时是否未错误扩散
5. 原文未出现个股是否标注 `映射补全`
6. 没有明确产业链映射依据时是否未补个股

## 5. 漏股自检

检查来源：

1. 涨价消息
2. 公告精选
3. 业绩预告
4. 涨停事件
5. 连板梯队
6. 热榜
7. 资金流向
8. 异动公告
9. 研报代表股
10. 小作文
11. 实盘赛文字
12. 游资复盘

## 6. CSV 校验

固定字段：

```csv
任务ID,股票代码,股票名称,所属方向,当日表现,核心逻辑浓缩,证据类型,舆情强度,情绪强度,兑现路径,验证点,负面触发,强度,状态分类,写入状态,写入时间,失败原因
```

检查：

1. 是否输出独立的 `02_个股逻辑提取表_YYYY-MM-DD_HHMMSS.csv`
2. 是否未再输出 Markdown 版个股表
3. CSV 内容是否未混入研报正文、HTML 结构或解释性段落
4. CSV 是否位于 `输出结果/个股总表/`
5. `任务ID` 是否位于第一列，且格式是否为 `ID-YYYYMMDD-序号`
6. `写入状态/写入时间/失败原因` 是否位于最后三列
7. `写入状态` 是否默认写 `未写入`
8. `写入时间` 是否默认留空
9. `失败原因` 是否默认留空

## 7. HTML 校验

检查：

1. 是否输出 `03_日内投研研报_YYYY-MM-DD_HHMMSS.html`
2. HTML 内容是否与 Markdown 研报保持一致
3. 是否具备清晰模块分区
4. 重点信息是否通过加粗或颜色增强可读性
5. 页面是否适合直接阅读
6. HTML 是否为独立文件，而非内嵌在 Markdown 中
7. HTML 是否位于 `输出结果/HTML研报/`

## 8. 文件拆分校验

检查：

1. 是否同时输出 Markdown、CSV、HTML 三个文件
2. 三个输出是否分别落在三个不同文件夹中
3. Markdown 研报是否位于 `输出结果/Markdown研报/`
4. CSV 个股总表是否位于 `输出结果/个股总表/`
5. HTML 阅读版是否位于 `输出结果/HTML研报/`
6. `01_日内投研研报` 是否只承载 Markdown 研报
7. `02_个股逻辑提取表` 是否只承载 CSV 个股总表
8. `03_日内投研研报` 是否只承载 HTML 阅读版
